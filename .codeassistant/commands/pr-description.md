---
description: "Создание описания для Pull Request"
---

1. Получить diff текущей ветки по сравнению с веткой master во временный файл:

    ```bash
    mkdir -p tmp
    git diff main > tmp/changes.diff
    ```

2. На основе анализа `changes.diff` создать в папке файл `pr_description.md` в формате Markdown с описанием изменений. Требования к описанию:
    - Имеется общее описание изменений.
    - Конкретно что изменилось: какие объекты были созданы, изменены, удалены.
    - Не пытаться объяснять каждое изменение, достаточно указать, что было изменено.
    - Если изменение касается нескольких объектов, то оно должно быть указано в одном описании.
    - Правила оформления документа должны соответстовать правилам markdownlint.

3. Проверить созданный файл при помощи markdownlint-cli и исправить найденные замечания.

4. Удалить `changes.diff`
