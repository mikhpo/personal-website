# Инструкции для ИИ-агентов

Этот документ содержит инструкции для ИИ-агентов, работающих с проектом персонального сайта. Проект разработан как шаблон для создания других сайтов, поэтому код, функции и классы максимально абстрактны.

## Общая информация о проекте

Проект представляет собой персональный сайт на Django с модульной архитектурой. Основные характеристики:

- Бэкенд: Python/Django
- Фронтенд: HTML/CSS с Bootstrap
- База данных: PostgreSQL
- Контейнеризация: Docker
- Тестирование: pytest

## Архитектура проекта

```text
personal-website/
├── personal_website/          # Основной Django-проект
│   ├── accounts/             # Управление аккаунтами пользователей
│   ├── blog/                 # Функциональность блога
│   ├── gallery/              # Галерея фотографий
│   ├── main/                 # Главная страница и основные компоненты
│   └── personal_website/     # Настройки проекта
├── docs/                     # Документация
├── static/                   # Статические файлы
├── tests/                    # Тесты
└── nginx/                    # Конфигурация Nginx
```

## Рекомендации по работе с кодом

### При работе с кодом

1. Все Django-приложения следуют стандартной структуре:
   - models.py - модели данных
   - views.py - обработчики запросов
   - urls.py - маршрутизация
   - forms.py - формы и валидация
   - templates/ - HTML-шаблоны
   - static/ - статические файлы приложения
   - tests/ - тесты

2. При генерации кода следуйте существующим паттернам и стилю:
   - Используйте type hints
   - Пишите docstrings для функций и классов
   - Соблюдайте PEP 8
   - Используйте Django ORM правильно

3. При создании новых функций:
   - Определите, в какое существующее приложение лучше вписать функционал
   - Если нужен новый модуль - создайте Django app
   - Реализуйте полноценное покрытие тестами

### При анализе задач

1. Перед началом работы:
   - Ознакомьтесь с README.md и этой документацией
   - Изучите структуру целевого приложения
   - Поймите, как связаны различные компоненты

2. При решении задач по функциональности:
   - Ищите аналогичные реализации в проекте
   - Анализируйте модели данных
   - Понимайте поток выполнения запроса

3. При написании кода:
   - Сохраняйте абстрактность и обобщенность решений
   - Учитывайте возможность переиспользования кода
   - Поддерживайте консистентность с остальным проектом

### При тестировании

1. Используйте pytest для запуска тестов:

   ```bash
   python -m pytest
   ```

2. Пишите тесты для нового функционала:
   - Модульные тесты для отдельных функций
   - Интеграционные тесты для взаимодействия компонентов
   - Тесты представлений и моделей

### При работе с документацией

1. Все документы ведутся на русском языке
2. Соблюдайте структуру Markdown
3. Используйте понятные заголовки и списки
4. Приводите примеры кода с пояснениями

## Особенности проекта

1. Проект является шаблоном для других сайтов:
   - Все решения должны быть максимально абстрактны
   - Избегайте жестко закодированных значений
   - Предусматривайте возможность кастомизации

2. Безопасность:
   - Всегда проверяйте пользовательский ввод
   - Используйте встроенные механизмы Django для защиты
   - Будьте внимательны к XSS и SQL injection

3. Миграции БД:
   - При изменении моделей создавайте миграции:

     ```bash
     python personal_website/manage.py makemigrations
     python personal_website/manage.py migrate
     ```

## Полезные команды

- Запуск сервера разработки:

  ```bash
  python personal_website/manage.py runserver
  ```

- Запуск тестов:

  ```bash
  python -m pytest
  ```

- Создание нового приложения:

  ```bash
  python personal_website/manage.py startapp app_name
  ```

- Проверка кода:

  ```bash
  ruff check .
  mypy .
  ```
